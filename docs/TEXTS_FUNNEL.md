# Тексты воронки — сообщения и кнопки по этапам

Все тексты, которые видит пользователь в сообщениях бота и на кнопках.

**Клавиатуры:** используются только Inline-кнопки (InlineKeyboard); ReplyKeyboard не используется. Обработчики — `callback_query`.

---

## 1. Старт (главное меню)

**Шаг:** `main`  
**Триггер:** команда `/start` или возврат «Назад» с других экранов.

**Сообщение:**
Neuro Option — система готовых торговых решений на основе анализа объёмов.

Вы получаете:

📍 готовые точки входа
📊 структурированные сигналы
🔄 регулярные обновления

Чтобы продолжить — нажмите «Получить доступ».

**Кнопки:**
- `🚀 Получить доступ`
- `Поддержка`

---

## 2. Получить доступ — выбор статуса аккаунта

**Шаг:** `choose_account_status`  
**Триггер:** нажатие «Получить доступ».

**Сообщение:**
Мы работаем с брокером **Intrade Bar**.

⚙️ Алгоритм системы настроен под эту площадку.

У вас уже есть аккаунт?

**Кнопки:**
- `✅ Да, есть аккаунт`
- `📝 Нет, зарегистрироваться`
- `⬅️ Назад`

---

## 3a. Нет аккаунта — регистрация по ссылке

**Шаг:** остаётся `choose_account_status`  
**Триггер:** нажатие «Нет, зарегистрироваться».

**Сообщение:**
Чтобы подключиться к системе, зарегистрируйтесь по нашей партнёрской ссылке:

🔗 {affiliate_link}

⏱ Регистрация занимает 1–2 минуты.

Доступ к торговым решениям предоставляется бесплатно.

⬅️ После регистрации вернитесь в бот и нажмите «✅ Я зарегистрировался».

**Кнопки:**
- `✅ Я зарегистрировался`
- `⬅️ Назад`

---

## 3b. Ожидание ID после новой регистрации

**Шаг:** `awaiting_new_registration_id`  
**Триггер:** нажатие «Я зарегистрировался».

**Сообщение:**
> Отправьте ваш ID аккаунта (числом).

**Кнопки:** `get_await_id_keyboard()` (например «Назад»).

**При ошибке (не число):**
> ❗ID должен быть числом. Попробуйте ещё раз.

**После успешной отправки ID:**
> ✅ ID получен. Мы проверим его и свяжемся с вами.

**Кнопки:**
- `Поддержка`

---

## 3c. Есть аккаунт — ввести ID или переподключить

**Шаг:** `existing_account_options`  
**Триггер:** нажатие «Да, есть аккаунт».

**Сообщение:**
Ваш аккаунт Intrade Bar зарегистрирован по нашей партнёрской ссылке?

**Кнопки:**
- `✅ Да — ввести ID`
- `⚙️ Нет — переподключить`
- `⬅️ Назад`

---

## 3d. Ожидание ID существующего аккаунта

**Шаг:** `awaiting_existing_id`  
**Триггер:** нажатие «Ввести ID» (из экрана 3c или из экрана 3e).

**Сообщение:**
- Из экрана 3c: `AWAIT_ID_TEXT` — «Отправьте ваш ID аккаунта (числом).»
- Из экрана 3e (переподключение): `AWAIT_RECONNECT_ID_TEXT` — другой текст.

**Кнопки:** `get_await_id_keyboard()` (только «Назад»).

**При ошибке (не число):**
>❗ID должен быть числом. Попробуйте ещё раз.

**После успешной отправки ID:**
> ✅ ID получен. Мы проверим его и свяжемся с вами.

**Кнопки:**
- `Поддержка`

---

## 3e. Запрос на переподключение

**Шаг:** `reconnect_instruction`  
**Триггер:** нажатие «Нет — переподключить».

**Сообщение:**
Если аккаунт уже создан, его можно перевести под нашу партнёрскую ссылку.

Для этого напишите в поддержку **Intrade Bar**
👇 Нажмите на текст, чтобы скопировать его.

```
Здравствуйте, переведите меня пожалуйста по партнёрской ссылке
{affiliate_link}
```

После ответа поддержки вернитесь сюда и отправьте ваш ID аккаунта.

**Кнопки (только Inline):**
- `⌨️ Ввести ID` — переход в ожидание ID
- `⬅️ Назад` — возврат к экрану «Ввести ID / Помогите переподключить»
- `Поддержка`

---

## 4. Поддержка

**Шаг:** `awaiting_support_message`  
**Триггер:** нажатие «Поддержка».

**Сообщение:**
> Напишите ваше сообщение — мы ответим в ближайшее время.

**Кнопки:** нет (пользователь вводит текст).

**После отправки сообщения:**
> Сообщение отправлено.

*(Возврат в главное меню.)*

## Возврат «Назад»

| Откуда | callback_data | Куда |
|--------|---------------|------|
| «У вас уже есть аккаунт?» (2) | `account:back` | Старт (главное меню) |
| Экран «Нет, зарегистрироваться» (3a) | `register:back` | «У вас уже есть аккаунт?» (2) |
| «Есть аккаунт» (3c) | `existing:back` | «У вас уже есть аккаунт?» (2) |
| «Запрос на переподключение» (3e) | `reconnect:back` | «Есть аккаунт» (3c) |
| Ожидание ID (3b, 3d) | `await_id:back` | Зависит от контекста: 3a, 3c или 3e |

## Сводка callback_data (keyboards → handlers)

| callback_data | Обработчик | Файл |
|---------------|------------|------|
| `main:get_access` | handle_get_access | start.py |
| `main:support` | handle_support | start.py |
| `account:yes` | handle_yes_existing_account | registration.py |
| `account:no` | handle_no_register | registration.py |
| `account:back` | handle_back_from_registration | registration.py |
| `register:done` | handle_registered_click | registration.py |
| `register:back` | handle_back_from_no_register | registration.py |
| `existing:enter_id` | handle_enter_id | registration.py |
| `existing:reconnect` | handle_reconnect_request | registration.py |
| `existing:back` | handle_back_from_existing_account | registration.py |
| `reconnect:enter_id` | handle_reconnect_enter_id | registration.py |
| `reconnect:back` | handle_reconnect_back | registration.py |
| `await_id:back` | handle_await_id_back | registration.py |

Префиксы уникальны: `main:`, `account:`, `register:`, `existing:`, `reconnect:`, `await_id:` — конфликтов нет.

## Уведомления в админ-чат

Сообщения в `ADMIN_CHAT_ID` — единый формат: emoji + заголовок, блок пользователя, Broker ID (если есть), блок «Действие».

| Тип | Emoji | Заголовок |
|-----|-------|-----------|
| Новая регистрация | 🆕 | Новая регистрация |
| Существующий аккаунт | 👤 | Существующий аккаунт |
| Запрос на переподключение | 🔁 | Запрос на переподключение |
| Сообщение от пользователя | 💬 | Сообщение от пользователя |

Блок пользователя: `Пользователь: @username` или `Пользователь: Без username` (Telegram ID не отображается).
